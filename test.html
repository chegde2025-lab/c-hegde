SET Processing Architecture – Existing vs Proposed Design

Existing Design Architecture

Flow Summary:
1. BLCC IDMC process (scheduled Daily / Weekly / Monthly):
  a) Updates SET_T and SET_ENTRY tables hosted in Oracle database.
  b) Sends set_notification input file to FDR S3 with set id details.
2. CFLI Cloud parses set_notification file and updates HISTORICAL_SNAPSHOT table.
3. IDMC workflows read HISTORICAL_SNAPSHOT for unprocessed rows and process SET XML from SET_ENTRY in BLCC.

Proposed AWS-Based Design

Flow Summary:
1. AWS EventBridge schedules batch job (Daily / Weekly / Monthly).
2. AWS Batch Python job runs inside Docker container.
3. SET XML (OpenReserve, CloseReserve) is generated.
4. Batch updates SET_T, SET_ENTRY and SET_NOTIFICATION (renamed from HISTORICAL_SNAPSHOT) with set id details.
5. IDMC reads SET_NOTIFICATION for unprocessed rows and processes SET XML from SET_ENTRY hosted in AWS cloud.
